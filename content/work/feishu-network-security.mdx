## 背景

很多飞书的安全功能要分"内网/外网"分别配置：在内网放开，在外网限制。
但前提是：**飞书得知道"什么是这家公司的内网"**。

这个配置一年管理员可能只点一次，但点错了，所有依赖它的安全策略都失效。
所以它必须：**能让没接触过这块的管理员，第一次就配对**。

## 我做了什么

### 步骤化的服务状态

把整个配置变成一条三段流程：
**反向代理服务器地址 → 内网凭证 → 内网识别服务**。
每一段单独显示当前状态（已配置 / 未配置 / 未验证）和对应操作（编辑 / 验证）。

管理员一眼看到自己卡在哪一步——配置中间态不再是黑盒。

### 凭证的过期可视化

内网凭证有效期不长，过期后服务就失效。
- 顶部 banner：「有 N 个内网凭证已过期，请尽快创建」
- 表格里过期项的「过期时间」用红字显示
- 操作列保留"下载 / 启用"——过期了也允许查看历史

避免静默失效造成的故障。

## 难点

- 这是一个**高后果、低频率**的设置项；既不能简化到不严肃，也不能堆叠技术细节。
- 反向代理服务器地址、凭证指纹这些概念对非技术管理员陌生；用"了解更多"链接出文档，正文保持业务语言。
